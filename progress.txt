## GunCompare Project – Current Status (Django Prototype)

### ✅ What I’ve Built So Far

I have a working Django web app called **gun_site** with an app called **rifles**.

Core functionality implemented:

* Django project + app structure is set up correctly
* Admin panel is working
* I can manually add rifle data through `/admin`
* Homepage (`/`) shows a list of rifles
* Clicking a rifle goes to a **detail page**
* I can select multiple rifles and use a **compare feature**

---

### 🧱 Current Project Structure

```
gun_site/
├── manage.py
├── gun_site/

│

│           ├── compare.html
```

---

### 🧠 Current Data Model

Basic Rifle model:

* manufacturer
* model
* caliber
* action
* weight_lbs
* barrel_length_in
* overall_length_in
* twist_rate
* msrp

Data is being entered via Django admin.

---

### 🔗 Routing Setup

Main URLs (`gun_site/urls.py`):

```python
path('', include('rifles.urls'))
```

App URLs (`rifles/urls.py`):

* `/` → rifle list
* `/rifle/<id>/` → detail page
* `/compare/` → comparison page

---

### 🔍 Features Working

#### 1. Rifle List Page

* Displays all rifles in database
* Each rifle links to detail page

#### 2. Rifle Detail Page

* Shows full specs for a single rifle

#### 3. Compare Feature

* Checkbox selection on list page
* Submits to `/compare/?ids=1&ids=2`
* Displays side-by-side comparison table
